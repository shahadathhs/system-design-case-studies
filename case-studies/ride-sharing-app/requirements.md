# Requirements

## Functional Requirements

*   Riders can request a ride from their current location to a destination.
*   Drivers can see nearby ride requests and accept them.
*   The system should match riders with the nearest available drivers.
*   Riders and drivers can see each other's real-time location on a map.
*   The system should calculate the fare for the ride.
*   Users can pay for their rides through the app.
*   Riders can rate their drivers and drivers can rate their riders.

## Non-Functional Requirements

*   **High Availability:** The service must be highly available, especially the ride-booking functionality.
*   **Low Latency:** Real-time location updates and ride matching must have low latency.
*   **Scalability:** The system should be able to handle a large number of concurrent riders and drivers.
*   **Consistency:** The system must maintain a consistent state, especially for ride status and payments.
